ภาพรวมการพัฒนาโปรเจค (ฉบับปัจจุบัน - Node.js Performance Optimized)

เรากำลังพัฒนาเว็บแอปพลิเคชันสำหรับบันทึกข้อมูล "ใบรายงานการผลิต" ทั้ง 22 รูปแบบ เพื่อเปลี่ยนกระบวนการทำงานจากเดิมที่ใช้ Excel มาสู่ระบบดิจิทัลที่มีประสิทธิภาพสูงสุด

Phase 1: สถาปัตยกรรมและฐานข้อมูล (Architecture & Database)
	1. ออกแบบสถาปัตยกรรม (Architecture Design):
		○ Frontend (React + Vite): รับผิดชอบการสร้าง UI และ Logic ของฟอร์มทั้ง 22 รูปแบบ
		○ Backend (Node.js Optimized): ใช้ Node.js ที่ปรับจูนประสิทธิภาพสูงสุด รองรับการเชื่อมต่อ Parallel และ Caching
		○ Database (SQL Server): ออกแบบ Table แบบ Hybrid (Relational + JSON) รองรับข้อมูลที่หลากหลาย และมีการทำ Indexing เต็มรูปแบบ
	2. ฐานข้อมูล (Database):
		○ ตารางหลัก: Form_Submissions, Form_Submission_Data, Form_Version_Sets

Phase 2: การพัฒนา Backend (API & Performance)
	1. Runtime & Server:
		○ ใช้ **Node.js** เป็นแกนหลักในการรัน Server
		○ ใช้ **Express JS** เป็น Web Server Framework
	2. Performance Features (จุดเด่นระบบ - แม้ใช้ Node.js ก็เร็ว):
		○ **In-Memory Caching:** ระบบจำข้อมูล Master Data (Template) ลงใน RAM ทำให้การดึงข้อมูลซ้ำๆ ทำได้ในเวลา 0ms
		○ **Warm Browser (PDF):** ระบบเปิด Browser รอไว้เบื้องหลัง (Singleton Pattern) ทำให้การกดพิมพ์ PDF ทำได้ทันที (< 1 วินาที)
		○ **Async Logging (Pino):** ระบบบันทึก Log แบบ Asynchronous ลดภาระการทำงานของ CPU
		○ **Database Indexing:** สร้างสารบัญ (Index) ให้กับคอลัมน์สำคัญ (Status, Lot No, Date) ทำให้ค้นหาและกรองข้อมูลได้รวดเร็ว
	3. APIs หลัก:
		○ Authentication, Submission Management, Reporting

Phase 3: การพัฒนา Frontend (UI/UX)
	1. Tech Stack: React 18, Tailwind CSS, Vite, Axios
	2. Form Generation Engine: ระบบ Render ฟอร์มอัตโนมัติตาม Config
	3. Report & Dashboard: หน้าแสดงรายการที่ทำงานร่วมกับ DB Indexing ได้อย่างลื่นไหล

Phase 4: Security & Deployment
	1. Security: CORS Policy, Env Variables
	2. Deployment: รองรับ Windows Server (Node.js) และการจัดการ Process ด้วย PM2
